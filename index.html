ПУТЬ OTRS (Рекомендуемый) ⭐

Оптимальное решение

Ключевое преимущество

Единая точка интеграции: Приказ на увольнение в ОКА становится триггером для автоматического запуска всех процессов увольнения.

Что НЕ требует разработки

  • Соглашение сторон
  • По медицинским основаниям
  • По случаю смерти работника/работодателя (Ст. 83)
  • По дисциплинарным основаниям
  • Сокращение штата
  • Истечение срока трудового договора

Как это работает

  1. Весь кадровый документооборот переходит из ЕСЭД в OTRS
  2. Приказ на увольнение в ОКА запускает автоматический процесс
  3. OTRS автоматически рассылает оповещения всем службам
  4. Централизованное управление всеми видами увольнений

Преимущества

Быстро: Экономия 12-18 месяцев разработки
Экономично: Снижение затрат в 6 раз
Централизовано: Всё в одной системе
Автоматизировано: Автооповещения через OTRS

Итоговый результат

Все 6 видов увольнения работают автоматически через единый триггер без необходимости разработки каждого вида отдельно в HR-сервисах!

ПУТЬ HRS (Долгий путь)

Требует значительных ресурсов

Основная проблема

Каждый вид увольнения требует отдельной разработки в HR-сервисах. Это означает 6 параллельных проектов с собственными циклами разработки, тестирования и внедрения.

Что требует разработки

  1. Соглашение сторон - отдельный модуль
  2. По медицинским основаниям - отдельный модуль
  3. По случаю смерти - отдельный модуль
  4. По дисциплинарным основаниям - отдельный модуль
  5. Сокращение штата - отдельный модуль
  6. Истечение срока договора - отдельный модуль

Временные затраты

Общий срок реализации: 12-18+ месяцев

  • Разработка каждого модуля: 2-3 месяца
  • Тестирование: 1-2 месяца
  • Интеграция: 2-4 месяца
  • Внедрение и поддержка: постоянно

Недостатки

Дорого: × 6 стоимость разработки
Долго: 12-18+ месяцев
Сложно: Множество интеграций
Ресурсоёмко: Большая команда

Итоговый результат

Каждый вид увольнения требует индивидуальной разработки, тестирования и внедрения, что приводит к значительным временным и финансовым затратам.

Сценарий 5: OTRS

Ключевое решение

  • Снижение затрат на разработку
